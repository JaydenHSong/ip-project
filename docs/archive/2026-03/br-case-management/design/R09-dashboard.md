# R09: BR 케이스 대시보드/통계

> **중요도**: ★★★☆☆ (중간)
> **난이도**: ★★★☆☆ (중간)
> **Phase**: 3
> **의존성**: R5 (이벤트 데이터), R11 (모니터링 데이터)
> **병렬 가능**: ⚠️ Phase 3, 충분한 데이터 축적 후 의미 있음

---

## 1. 문제

BR 신고의 전체 성과를 파악할 수 없음:
- 해결률이 얼마인지
- 평균 응답 시간이 어떤지
- 어떤 위반 유형이 잘 해결되는지
- 트렌드가 좋아지고 있는지

## 2. 솔루션 (Zendesk SLA Report + Freshdesk Dashboard 참조)

### 2.1 주요 KPI 카드

```
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
│ 총 케이스│ │ 해결률  │ │ 평균응답│ │ SLA 준수│
│   127   │ │  68%   │ │  3.2일  │ │  82%   │
│ +12 이번주│ │ ↑5%   │ │ ↓0.5일 │ │ ↑3%   │
└────────┘ └────────┘ └────────┘ └────────┘
```

### 2.2 차트

| 차트 | 유형 | 설명 |
|------|------|------|
| 케이스 추이 | Line | 주간 신규/해결/미해결 추이 |
| 위반유형별 해결률 | Bar | 카테고리별 해결 비율 |
| 평균 응답 시간 | Bar | 카테고리별 아마존 응답 시간 |
| 상태 분포 | Donut | 현재 활성 케이스 상태 분포 |
| SLA 준수율 추이 | Line | 주간 SLA 준수/위반 비율 |

### 2.3 필터

- 기간: 1주/1개월/3개월/전체
- 위반 카테고리
- 마켓플레이스

## 3. 구현 범위

### 3.1 API
- `GET /api/dashboard/br-case-stats` — KPI 카드 데이터
- `GET /api/dashboard/br-case-trends` — 차트 데이터 (기간별)
- `GET /api/dashboard/br-case-by-category` — 카테고리별 통계

### 3.2 UI
- Dashboard 위젯으로 추가 또는 별도 페이지 (`/reports/br-dashboard`)
- 기존 차트 라이브러리(Recharts) 재사용
- 기존 `chart-colors.ts` 확장
