# Amazon User Behavior Personas (50+)

크롤러가 사람처럼 행동하려면 다양한 실제 유저 행동 패턴을 모방해야 한다.
아래는 아마존 쇼핑 시 관찰되는 50+ 유형의 사용자 행동 패턴이다.

## 행동 차원 분류

### A. 검색 행동 (Search Behavior) — 15 패턴

| # | 페르소나 | 행동 패턴 | 크롤러 적용 |
|---|---------|----------|-----------|
| A01 | 정확 검색자 | 브랜드+제품명 정확히 입력 ("spigen tough armor iphone 15") | 긴 키워드, 타이핑 속도 보통 |
| A02 | 막연 검색자 | 카테고리 수준 입력 ("phone case") | 짧은 키워드, 결과 많이 스크롤 |
| A03 | 오타 수정자 | 타이핑 중 백스페이스로 수정 | 중간에 Backspace 2~3회 |
| A04 | 자동완성 클릭자 | 2~3글자 입력 후 드롭다운에서 선택 | 타이핑 멈춤 → 드롭다운 클릭 |
| A05 | 복붙 검색자 | 다른 곳에서 키워드 복사 붙여넣기 | Ctrl+V 한번에 입력 (타이핑 없음) |
| A06 | 재검색자 | 결과 불만족 → 검색창 수정 → 재검색 | 검색창 클리어 → 새 키워드 |
| A07 | ASIN 검색자 | ASIN 직접 입력 ("B09V3KXJPB") | 영숫자 10자, 빠른 타이핑 |
| A08 | 느린 타이핑자 | 한 글자당 200~500ms | 고령자/모바일 유저 패턴 |
| A09 | 빠른 타이핑자 | 한 글자당 30~80ms | 숙련자 패턴 |
| A10 | 카테고리 브라우저 | 검색 안 하고 카테고리 메뉴 클릭 | 홈 → 메뉴 → 카테고리 순 |
| A11 | 필터 사용자 | 검색 후 브랜드/가격/평점 필터 클릭 | 사이드바 필터 클릭 |
| A12 | 정렬 변경자 | "Price: Low to High" 등 정렬 변경 | 드롭다운 클릭 |
| A13 | 2단어 검색자 | "spigen case" 같은 2단어 | 가장 흔한 패턴 |
| A14 | 긴 문장 검색자 | "best phone case for iphone 15 pro max drop protection" | 긴 문장, 느린 타이핑 |
| A15 | 음성 검색자 | 마이크 버튼 클릭 (웹에서는 드뭄) | 거의 사용 안 함 |

### B. 스크롤 행동 (Scroll Behavior) — 10 패턴

| # | 페르소나 | 행동 패턴 | 크롤러 적용 |
|---|---------|----------|-----------|
| B01 | 빠른 스크롤러 | 결과를 빠르게 훑고 끝까지 내림 | 200~300px씩 빠르게 |
| B02 | 천천히 읽는 자 | 각 상품을 멈춰서 읽음 | 상품 하나당 1~3초 멈춤 |
| B03 | 위아래 왕복자 | 스크롤 다운 → 다시 올라가서 비교 | 하단 → 상단 → 중간 |
| B04 | 페이지 하단 안 보는 자 | 상위 5~10개만 보고 클릭 | 30% 이하만 스크롤 |
| B05 | 끝까지 보는 자 | 페이지 끝까지 스크롤 후 다음 페이지 | 100% 스크롤 → Next 클릭 |
| B06 | 점프 스크롤러 | 스크롤바 드래그로 점프 | 한번에 큰 거리 이동 |
| B07 | 마우스휠 스크롤러 | 마우스 휠로 부드럽게 스크롤 | 일정 간격 연속 스크롤 |
| B08 | 트랙패드 스크롤러 | 관성 스크롤 (Mac) | 빠르게 시작 → 점차 감속 |
| B09 | Page Down 유저 | 키보드 PageDown 사용 | 화면 단위 이동 |
| B10 | 스페이스바 스크롤러 | 스페이스바로 페이지 다운 | 화면 단위 이동 |

### C. 상품 클릭 행동 (Click Behavior) — 10 패턴

| # | 페르소나 | 행동 패턴 | 크롤러 적용 |
|---|---------|----------|-----------|
| C01 | 이미지 클릭자 | 상품 이미지를 클릭해서 상세 진입 | img 요소 클릭 |
| C02 | 제목 클릭자 | 상품명 텍스트를 클릭 | h2 a 요소 클릭 |
| C03 | 새 탭 오픈자 | Ctrl+클릭 또는 우클릭 → 새 탭 | 새 탭에서 열기 |
| C04 | 첫 결과 클릭자 | 항상 첫 번째 결과 클릭 | position 1 클릭 |
| C05 | 비교 쇼핑자 | 여러 상품 번갈아 클릭 → 뒤로가기 | 클릭 → 뒤로 → 다음 클릭 |
| C06 | 가격 먼저 보는 자 | 가격 영역에 마우스 호버 후 클릭 | 가격 요소 근처 마우스 이동 |
| C07 | 리뷰 수 보는 자 | 별점/리뷰 수 확인 후 결정 | 리뷰 영역 호버 → 클릭 |
| C08 | 스폰서 무시자 | 스폰서 상품 건너뛰고 오가닉 결과 클릭 | 첫 2~4개 스킵 |
| C09 | 스폰서 클릭자 | 스폰서 상품도 자연스럽게 클릭 | 무작위로 스폰서 포함 |
| C10 | 뒤로가기 유저 | 상세 보고 뒤로가기로 목록 복귀 | page.goBack() 사용 |

### D. 상세 페이지 행동 (Detail Page Behavior) — 10 패턴

| # | 페르소나 | 행동 패턴 | 크롤러 적용 |
|---|---------|----------|-----------|

| D02 | 불릿 포인트 리더 | 특징 목록 천천히 읽음 | 불릿 영역에서 3~5초 머묾 |
| D03 | 리뷰 스크롤러 | 리뷰 섹션까지 스크롤하여 읽음 | 80%까지 스크롤 |

| D05 | 빠른 판단자 | 제목+가격+이미지만 보고 판단 | 3초 이내 스크롤 최소 |

### E. 세션 행동 (Session Behavior) — 8 패턴

| # | 페르소나 | 행동 패턴 | 크롤러 적용 |
|---|---------|----------|-----------|

| E02 | 장기 브라우저 | 30분+ 여러 검색 연속 | 긴 세션, 중간 휴식 |
| E03 | 점심시간 쇼핑 | 정오~1시 사이 짧은 세션 | 시간대별 행동 차이 |
| E04 | 야간 쇼핑 | 밤 9~11시 긴 세션 | 느리고 꼼꼼한 패턴 |
| E05 | 주말 쇼핑 | 토일 여유롭게 많은 상품 비교 | 많은 클릭, 긴 체류 |
| E06 | 멀티탭 유저 | 관심 상품 여러 탭으로 열어놓고 비교 | 3~5개 탭 동시 |

## 크롤러 적용 전략

### 세션별 랜덤 페르소나 조합

각 크롤링 세션(캠페인 1회)마다 아래 차원에서 랜덤으로 하나씩 조합하여 "가상 인격"을 생성한다:

```typescript
type CrawlPersona = {
  searchStyle: 'A01' | 'A02' | 'A03' | 'A08' | 'A09' | 'A13' | 'A14'
  scrollStyle: 'B01' | 'B02' | 'B04' | 'B05' | 'B07'
  clickStyle: 'C01' | 'C02' | 'C05' | 'C08'
  detailStyle: 'D01' | 'D02' | 'D03' | 'D05'
  sessionStyle: 'E01' | 'E02'
  typingSpeed: { min: number; max: number }  // ms per char
  scrollSpeed: 'slow' | 'medium' | 'fast'
  dwellTime: { min: number; max: number }    // ms on each product
  productsPerPage: number                     // 몇 개 보고 다음 페이지
}
```

### 가중치 분포 (현실 반영)

| 행동 | 비율 | 설명 |
|------|------|------|
| 제목 클릭 vs 이미지 클릭 | 60:40 | 제목 클릭이 더 흔함 |
| 빠른 타이핑 vs 느린 타이핑 | 30:70 | 대부분 보통~느림 |
| 스폰서 무시 vs 클릭 | 70:30 | 대부분 스폰서 건너뜀 |
| 첫 페이지만 vs 다음 페이지 | 60:40 | 과반이 1페이지만 봄 |
| 빠른 스크롤 vs 천천히 | 40:60 | 과반이 천천히 읽음 |
| 뒤로가기 vs 새 탭 | 70:30 | 뒤로가기가 주류 |
| 이미지 갤러리 보기 | 50% | 절반은 다른 이미지 확인 |
| 리뷰 섹션까지 스크롤 | 40% | 4할이 리뷰 확인 |
| 오타 수정 | 15% | 가끔 백스페이스 |
| 검색어 수정 재검색 | 20% | 5번 중 1번은 재검색 |

### 핵심 구현 파라미터

```typescript
// 타이핑 속도 프로파일
const TYPING_PROFILES = {
  slow:    { charDelay: [150, 400], pauseProb: 0.1, typoProb: 0.08 },
  medium:  { charDelay: [60, 180],  pauseProb: 0.05, typoProb: 0.05 },
  fast:    { charDelay: [25, 80],   pauseProb: 0.02, typoProb: 0.02 },
} as const

// 스크롤 속도 프로파일
const SCROLL_PROFILES = {
  slow:   { pixelsPerStep: [50, 120],  stepDelay: [100, 300] },
  medium: { pixelsPerStep: [100, 250], stepDelay: [50, 150] },
  fast:   { pixelsPerStep: [200, 400], stepDelay: [20, 80] },
} as const

// 체류 시간 (상품 카드 위에서 머무는 시간)
const DWELL_PROFILES = {
  scanner:  { min: 500, max: 1500 },   // 빠르게 훑기
  reader:   { min: 2000, max: 5000 },  // 꼼꼼히 읽기
  comparer: { min: 3000, max: 8000 },  // 비교 쇼핑
} as const

// 검색 → 상세 → 뒤로가기 딜레이
const NAVIGATION_DELAYS = {
  homeToSearch: [1500, 4000],       // 홈 도착 → 검색 시작
  searchToClick: [2000, 6000],      // 결과 로드 → 첫 클릭
  detailToBrowse: [3000, 15000],    // 상세 페이지 체류
  backToNextClick: [1000, 3000],    // 뒤로가기 → 다음 클릭
  betweenPages: [2000, 5000],       // 페이지 이동 간격
} as const
```

## AI 개입 지점과 페르소나 연동

| 상황 | 기존 | 개선 |
|------|------|------|
| 홈페이지 로드 | URL 직접 이동 (안 함) | 홈 접속 → AI가 정상 확인 |
| 검색창 찾기 | 안 함 (URL 직접 이동) | 셀렉터 시도 → 실패 시 AI가 스크린샷에서 찾기 |
| 검색 결과 파싱 | CSS 셀렉터만 | 셀렉터 → 실패 시 AI Vision |
| 상세 페이지 진입 | URL 직접 이동 | 검색 결과에서 클릭 (페르소나에 따라 이미지/제목) |
| 상세 페이지 파싱 | CSS 셀렉터만 | 셀렉터 → 실패 시 AI Vision |
| 차단 감지 | CAPTCHA만 | AI가 모든 차단 유형 판단 |
| 다음 페이지 이동 | URL 파라미터 변경 | Next 버튼 클릭 (페르소나에 따라) |
| 상세 → 목록 복귀 | 새 URL 이동 | 뒤로가기 (page.goBack) |
